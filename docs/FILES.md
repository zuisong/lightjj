# File Guide

Per-file map of the codebase: what each file does plus the invariants, gotchas, and design notes that matter when changing it. The compact one-line-per-file index lives in [CLAUDE.md](../CLAUDE.md#project-structure); this is the detailed version. Read a file's entry before modifying the file or adding a new caller of its exports; update the entry when adding, removing, or renaming a file or when its contract changes (and keep the CLAUDE.md one-liner in sync). Test files get entries only when the test setup itself carries invariants worth noting.

See [ARCHITECTURE.md](ARCHITECTURE.md) for the system-level design these files implement.

## cmd/lightjj/

- **`main.go`** ŌĆö CLI entry point, flag parsing, embeds frontend-dist/. Startup order matters for persistence: MigrateConfigIfNeeded ŌåÆ MigrateStateIfNeeded ŌåÆ tab restore (ReadPersistedTabs) ŌĆö the state migration must run before tab restore or upgraded users lose their tabs on first launch.
- **`frontend_embed.go`** ŌĆö `-tags embed` build: frontendHandler serves the go:embed'd frontend-dist/.
- **`frontend_stub.go`** ŌĆö Default (no embed tag) build: frontendHandler returns a static "frontend not bundled" help page pointing at release binaries or a `-tags embed` source build, so plain `go install` binaries don't 404.
- **`session_file.go`** ŌĆö Agent port-discovery: writes {pid,addr,repo_dir,mode} to `$XDG_RUNTIME_DIR/lightjj/sessions/<pid>.json` after bind; stale-pid sweep; symlink-race-hardened TempDir fallback (verifyOwnedDir double-check). No-op on Windows. Agents match by repo_dir instead of needing the random port.
- **`session_file_unix.go`** ŌĆö verifyOwner: checks the session dir is owned by the current uid (unix builds).
- **`session_file_other.go`** ŌĆö verifyOwner no-op stub for non-unix builds (Windows short-circuits earlier in writeSessionFile).
- **`api_cmd.go`** ŌĆö `lightjj api METHOD PATH [BODY]` + `lightjj sessions` subcommands: loopback-only HTTP client (validateAddr rejects non-localhost) with session-file discovery, for agent harnesses whose sandboxes denylist curl/wget. Security model + discovery algorithm in [design-notes/api-cli.md](design-notes/api-cli.md).
- **`skill_cmd.go`** ŌĆö `lightjj skill [install]`: prints or installs the go:embed'd SKILL.md (the agent-facing guide to `lightjj api`); install refuses to overwrite a file that doesn't carry the lightjj marker unless `--force`.
- **`apply_hunks.go`** ŌĆö `--apply-hunks=<spec.json>` re-entry path for `jj split --tool lightjj-hunks`: dumb writer applying the frontend-synthesized write/revert/delete spec into jj's $right dir, with path-escape checks. All hunk-patching logic lives in frontend hunk-apply.ts.

## internal/runner/

CommandRunner interface + implementations

- **`runner.go`** ŌĆö Interface definition (Run, RunWithInput, RunForMutation, StreamCombined, RunRaw, WriteFile)
- **`local.go`** ŌĆö LocalRunner: exec("jj", args) with configurable Binary. WriteFile does symlink-escape hardening (EvalSymlinks on parent + Lstat leaf) before os.WriteFile. ResolveWorkspaceRoot rejects forgotten workspaces: `jj workspace root` exits 0 inside a forgotten workspace's dir, so a `jj log -r @` probe (probeWorkspace) follows it ŌĆö tab create 400s and the startup restore loop log-and-skips instead of mounting a permanently broken tab.
- **`ssh.go`** ŌĆö SSHRunner: wraps jj args in ssh command. WriteFile pipes content via `cd <repo> && cat > <path>` over stdin ŌĆö no symlink check (same trust boundary as remote shell).

## internal/api/

HTTP handlers

- **`server.go`** ŌĆö Route registration, runMutation, op-id caching, workspace store reader, helpers. For pure decodeŌåÆvalidateŌåÆbuild mutations the route line IS the handler (`reg("POST /api/new", mutation(s, newRequest.validate, newRequest.build))`); the endpoint table in docs/ARCHITECTURE.md is drift-checked by architecture_doc_test.go. `Server.OpenTabRoots` (func field, injected by TabManager.addLocked; nil in tests/standalone) reports all open tab roots for cross-tab guards. The op-id cache is encapsulated behind `getOpId()`/`setOpId()`/`casOpId()` ŌĆö the `cachedOp`/`cachedMu` fields are an implementation detail nothing else (besides `refreshOpId`, which composes them with the actual jj read) should touch; the SSH poll loop's compare-and-swap goes through `casOpId`, never the mutex directly. `trySnapshot()` is the pause-respecting snapshot path used by snapshotLoop and handleFileWrite; handleSnapshot checks the pause inline instead because it needs runMutation's response envelope and runMutation itself bumps the pause counter.
- **`handlers.go`** ŌĆö Endpoint implementations, flag validation, and `mutation[Req](s, validate, build)` ŌĆö the generic decodeŌåÆvalidateŌåÆbuildŌåÆrunMutation factory. Pure mutations exist as request structs with `validate()`/`build()` method pairs (no named handler func); `opMutation`/`bookmarkMutation`/`bookmarkRevMutation`/`bookmarkRemoteMutation` are thin adapters over `mutation()` for families sharing one request shape. Hand-rolled handlers are the ones with extra behavior: stdin (describe), streaming (git push/fetch/alias/index-paths), runner-backed validation (alias), tempfiles (split-hunks, merge-resolve), non-400 guards (workspace forget), server-state builds (workspace add), post-mutation side effects (snapshot, file-write, unlock-repo). Every success response uses the MutationResult envelope `{output, warnings?}` (apiVersion 2 unified the legacy `{ok}`/`{status}` shapes). Workspace name validation is two-tier: `workspaceNameRe` (strict, ADD only ŌĆö name becomes a dir suffix) vs `validateWorkspaceRefName` (loose, forget/rename ŌĆö jj allows dots/slashes in existing names; flag injection blocked by the `--` in the command builders). `workspaceNameMutation(build, guard)` is the forget/rename factory (decodeŌåÆvalidateŌåÆguardŌåÆrunMutation; stays hand-rolled because guard returns non-400 statuses); `forgetOpenTabGuard` 409s when the target workspace's path (via `workspacePathMap`, shared with handleWorkspaces) matches any open tab root, and allows when the path can't be resolved.
- **`handlers_test.go`** ŌĆö Handler tests with MockRunner
- **`architecture_doc_test.go`** ŌĆö Drift guard for the docs/ARCHITECTURE.md endpoint table: parses `| METHOD | \`/api/...\` |` rows and asserts two-way sync with `routes()` (every registered route documented, every documented `/api` row registered). Tab routes (`/tabs`, `/tab/{id}/*`) are exempt ŌĆö they're TabManager's, not `routes()`'. Sibling of agent_docs_test.go (the agent_api.md guard).
- **`integration_test.go`** ŌĆö Integration tests (build-tagged)
- **`watcher.go`** ŌĆö fsnotify on .jj/repo/op_heads/heads/ + SSE push, periodic util snapshot. Internals are typed: broadcasts are `sseEvent{name, data, droppable}` values (constructors `opEvent`/`navEvent`/`pollFailEvent`, sentinel vars `evStaleWC`/`evFreshWC`/`evPollOk`) that map 1:1 onto the `event:`/`data:` wire frames ŌĆö the names/payload shapes are the api.ts contract and must not change; `handleEvents` is a single formatted write with zero event-type dispatch. `broadcast()` drops `droppable` events (op-ids, nav hints) when a subscriber's buffer is full, but EVICTS the oldest queued event to deliver non-droppable sentinels (stale-wc/fresh-wc, pollfail/pollok) ŌĆö they're edge-triggered and never re-sent, so a dropped one would stick the client until reconnect. Both background loops classify probe results through ONE shared `probeTracker` (`ok`/`staleWC`/`fail`): it owns the consecutive-error counter, the 3rd-then-every-12th log throttle, the `pollFailThreshold` UI gate, and the unconditional poll-fail-clear discipline (the stale-WC-interlude-leaves-pollFailed-stuck bug previously had to be fixed once per loop ŌĆö the invariant comments now live once, on the struct). SSH mode uses `sshPollLoop` (`PollOpId` ŌĆö implicit snapshot + op-id in one round trip; no remote deps; `Server.casOpId` CAS against the pre-poll cache value so a concurrent runMutation advance isn't regressed). `interval <= 0` disables both loops (parity). Stale-WC detection: `isStaleWCError()` ŌåÆ `setStale(bool)` (staleMu-serialized Swap+broadcast ŌĆö handler clear can't race a loop set and emit sentinels out-of-order; Swap edges don't re-fire so the race has no self-heal) ŌåÆ `evStaleWC`/`evFreshWC` ŌåÆ frontend non-dismissable warning with "Update stale" action. Both handleSnapshot + handleWorkspaceUpdateStale call `clearStale()`. Snapshot-pause counters are shared per canonical repo path via the package-level `pauseCounterFor` registry, resolved at watcher CONSTRUCTION (`pauseKey`: RepoDir locally, RepoPath over SSH) ŌĆö same-path tabs share one counter with no post-construction field swapping. `handleEvents` refreshes the op-id if the cache is empty (read-only session path), emits current stale + poll-fail state unconditionally on connect (both branches ŌĆö false branch covers SSE-dropped-during-staleness ŌåÆ CLI-recovered ŌåÆ reconnect), per-write WriteDeadline extension (not blanket disable ŌĆö dead TCP surfaces in Ōēż85s).
- **`watcher_test.go`** ŌĆö Broadcaster tests (subscribe/broadcast, droppable-drop on full buffer, sentinel eviction on full buffer), sshPollLoop tests via seqRunner (hook field for mid-call injection), setStale edge-only invariant, handleEvents refresh-if-empty table, CAS-guards-concurrent-advance, stale-interlude-clears-pollfail on BOTH loops (the shared probeTracker discipline), shared-pause-counter-by-path
- **`tabs.go`** ŌĆö TabManager: per-tab Server + Watcher mounted at /tab/{id}/; TabResolve + TabFactory injection (mode-agnostic validation via closure); openTabs persisted to **state.json** via `SetOpenTabs` (state.go ŌĆö NOT config.json, so tab churn never runs the JSONC patch machinery; tab 0 excluded ŌĆö fights with -R on next launch); startup restore re-runs resolve() so moved repos log-and-skip. `OpenTabRoots()` (RLock) returns every open tab's root; addLocked injects it into each mounted Server (feeds the workspace-forget cross-tab guard). NewTabManager also registers the host-scoped config + state routes on its Mux. addLocked wires only the SSE subscriber hooks (idle shutdown); snapshot-pause sharing across same-path tabs is NOT wired here ŌĆö watchers resolve their shared counter at construction (`pauseCounterFor`, watcher.go).
- **`config.go`** ŌĆö Server-side JSONC config (tailscale/hujson) for HUMAN-edited keys only; mergeAndWriteConfig applies per-key RFC-6902 add patches so comments on existing keys survive; fresh installs (ENOENT) seed from configTemplate, corrupt files NEVER reseed ŌĆö 422 + frontend warning keeps user's bad file untouched (ErrConfigUnparseable sentinel). Machine-written keys (openTabs, recentActions) live in state.json (state.go), never here. GET /api/config returns Standardized JSON (422 on corrupt); GET/POST /api/config/raw round-trips raw JSONC for ConfigModal (text/plain only, 1MB cap, isLocalOrigin guard). Atomic writes go through the shared `atomicWriteFile` (jsonstore.go).
- **`config_jsonc.go`** ŌĆö hujson helpers: standardizeJSONC (bytes.Clone first ŌĆö hujson.Standardize mutates input via aliased Extra slices), unmarshalJSONC, patchConfigKeys, removeConfigKeys (RFC-6902 "remove"; only pass keys that exist ŌĆö removing an absent path errors). The centralised "Standardize before Unmarshal" idiom ŌĆö any new json.Unmarshal on config bytes must go through unmarshalJSONC or break the first time the user adds a comment.
- **`config_template.go`** ŌĆö First-run JSONC template string (in-tree constant, not a file). Teaching comments over theme, fontSize, editorArgs, font families. Kept in sync with frontend defaults and [CONFIG.md](CONFIG.md) by convention ŌĆö see the top-of-file comment.
- **`state.go`** ŌĆö Machine-state store: state.json (plain JSON, NOT JSONC) next to config.json, holding values lightjj writes itself ŌĆö `openTabs` (PersistedTab; SetOpenTabs filter-merges by (mode,host) so concurrent sessions don't stomp each other; ReadPersistedTabs for startup restore) and `recentActions` (namespace ŌåÆ key ŌåÆ ms timestamp; SetRecentActions whole-section replace). One `stateMu` over every read-modify-write; atomic writes via `atomicWriteFile`. Missing/zero-byte/corrupt files all read as fresh (machine state is recoverable; nothing to protect with a 422). `MigrateStateIfNeeded` (main.go startup, after MigrateConfigIfNeeded, before tab restore) imports legacy openTabs/recentActions from config.json then removes them via removeConfigKeys ŌĆö import is written BEFORE removal so failure can't lose data. HTTP: GET/POST /api/state/recent-actions (package-level handlers, host-scoped like config; POST has the isLocalOrigin guard).
- **`jsonstore.go`** ŌĆö Generic `jsonCollection[T hasID]` keyed flat-file JSON-array store (key ŌåÆ one file via `pathFor`, items addressed by ID). Owns the machinery annotations + doc-comments previously hand-mirrored: per-collection read-modify-write mutex, merge-on-upsert (preserve-on-empty, runs BEFORE stamp so merge detects "client omitted X" by zero value), server-side `stamp` defaults, configurable `deleteMatch` (cascade), delete-file-when-last-item-removed, and `UpsertBatch` (whole batch under one lock). Collections MUST be package-level vars, not Server fields ŌĆö store files are shared across tabs, so the mutex must be too. Errors are `storeError{status,msg}`; handlers map via `storeErrStatus` (bad key ŌåÆ 400, I/O ŌåÆ 500). Low-level readJSONStore/atomicWriteJSON live here too, plus `atomicWriteFile` ŌĆö the SINGLE temp-file+rename primitive shared by jsonstore/config/state (it prevents torn writes only; lost-update prevention is the caller's mutex).
- **`annotations.go`** ŌĆö Per-changeId review comments (annotations/{changeId}.json): `Annotation` type + `annotationStore` collection config (pathFor=annotationsPath with changeId regex validation; merge preserves resolution/resolvedAtCommitId AND legacy Status under the same condition, plus createdAt; stamp fills createdAt) + thin GET/POST/DELETE handlers. DELETE without id = clear-all (DeleteAll).
- **`doc_comments.go`** ŌĆö Doc-mode range-anchored comments/suggestions: `DocComment`/`DocAnchor` types + `docCommentStore` collection config (key = RepoPath+"|"+cleanDocPath(filePath) via `docCommentKey`, filename = sha256(key)[:16]; merge preserves resolution/resolvedAt/createdAt; stamp fills randID + createdAt; deleteMatch cascades to replies via ParentId) + GET/POST/DELETE/batch handlers. cleanDocPath: ToSlash+Clean+strip ./, reject ../ and / (400). POST validates anchor.selection. DELETE requires id (no clear-all). Batch validates ALL comments before writing ANY (all-or-nothing via UpsertBatch).
- **`agent_docs.go`** ŌĆö `GET /api/agent` serves go:embed agent_api.md (the agent-facing contract). agent_docs_test.go fetches it and asserts every /api/* path mentioned is a registered route ŌĆö the structural guard against doc/route drift.
- **`symbol.go`** ŌĆö rg-backed go-to-definition for `GET /api/symbol` (Tier 1 ŌĆö heuristic, no language server): per-language definition regex run against the working copy via `rg --json` through RunRaw (so it works in SSH mode), identifier-gated `name` param, capped hits with leading doc-comment context.
- **`focus.go`** ŌĆö `GET/POST /api/focus`: frontend-reported "what is the user looking at" (change_id/commit_id/active view/doc path) for the agent steering loop. Server-stamped UpdatedAt, per-tab singleton (each tab is its own Server), field-length caps.
- **`open.go`** ŌĆö Open-in-$EDITOR: buildEditorArgv ({file}/{line} substitution, argv[0] validation), handleOpenFile. Setsid via build-tagged detachProcess (open_unix.go / open_windows.go). The ONE sanctioned `exec.Command` outside internal/runner/ ŌĆö the editor must spawn where lightjj runs (it opens a window on the user's machine), not where jj runs; routing through the runner would launch it on the SSH remote.
- **`open_unix.go`** ŌĆö detachProcess via Setsid (!windows) so Ctrl+C on lightjj doesn't kill the spawned editor.
- **`open_windows.go`** ŌĆö detachProcess no-op (windows builds).
- **`gzip.go`** ŌĆö Gzip response middleware (lazy-init writer, Flush passthrough for SSE)

## frontend/src/lib/

- **`api.ts`** ŌĆö Typed API client, op-id tracking, commit_id-keyed LRU cache. `wireAutoRefresh` SSE reconnects on backend restart via `everSawEvent` heuristic (mirrors Go's `everSawLine`): handleEvents pushes op-id immediately on connect, so CLOSED-before-event = watcher absent ŌåÆ give up; CLOSED-after-event = transient ŌåÆ backoff retry. `resolvedInfo()` exposes the memoized /api/info value synchronously (set when the `_info` promise resolves, cleared with the session memos) ŌĆö jj-features.svelte.ts reads the backend `features` map from it in the same tick App calls setDetectedJJVersion. All mutation methods return `MutationResult` (`{output, warnings?}`) ŌĆö including fileWrite/openFile/unlockRepo since the apiVersion-2 envelope unification. Machine-state methods `recentActions`/`saveRecentActions` (+ `RecentActionsState` type) hit /api/state/recent-actions ŌĆö host-scoped data, but routed through the normal tab-scoped request() since every Server registers the state routes. **Annotation client is namespaced** ŌĆö `api.annotations.{list,save,remove,clear}` mirrors `api.docComments.{list,upsert,remove}` (sibling review stores, one client shape); `api.annotations(changeId)` (bare call) and the flat `saveAnnotation`/`deleteAnnotation`/`clearAnnotations` are deprecated delegating aliases for pre-namespacing call sites (App/DiffPanel + their test mocks).
- **`RevisionGraph.svelte`** ŌĆö Revision list with graph gutter rendering; JS-tracked hoveredIndex (mousemove-driven, not :hover). ALWAYS windowed via `createWindower` (virtual.svelte.ts) ŌĆö no eager path, no threshold. Fixed 18px rows ŌåÆ pure-arithmetic windowing (no per-row measurement). `renderItems` falls back to a synthesized first-N-rows window (FALLBACK_ROWS) when the windower has no measured viewport yet ŌĆö the pre-effect first render in browsers, and permanently in jsdom (clientHeight=0), which is what makes component tests render rows at all. Inline-mode badges/previews are GENERIC over `ModeBase`: `activeMode` (the one active mode or null) + `sources`/`hasDestination` derive isSource/isTarget/showPreview; only label text dispatches through kind-keyed lookups (`sourceBadgeLabel`/`targetBadgeLabel`/`commandPreview`). Pure display ŌĆö no revset filter input (that lives in App.svelte above RevisionGraph inside `revision-panel-wrapper`). All context-menu emitters (row AND workspace badges) share the `anyModeActive || isRefreshing` suppression gate ŌĆö new right-click surfaces must add it too.
- **`virtual.svelte.ts`** ŌĆö `createWindower()` fixed-row-height virtualization. count/scrollEl passed as getters ŌåÆ reactive without $effect glue. `items` is $derived reading count directly ŌåÆ no lag-frame between count shrink and item re-render (the tanstack setOptions-via-$effect problem). `scrollToIndex` implements align:'auto'. `holdViewport(scrollEl, anchorSel, fn, gen?)` runs a layout-shifting mutation without the line under the user's eye moving: capture first `anchorSel` match with top Ōēź container-top ŌåÆ fn() ŌåÆ `await tick()` ŌåÆ correct scrollTop by delta (tick = post-DOM-flush pre-paint; rAF would paint shifted first). `anchorSel` MUST be per-row (e.g. `.diff-file`) ŌĆö a wrapper div's top doesn't move when its contents reflow. gen() lets a concurrent scrollTo win.
- **`GraphSvg.svelte`** ŌĆö SVG renderer for graph gutter characters (pipes, curves, node dots)
- **`DiffPanel.svelte`** ŌĆö Diff viewer: unified/split toggle, syntax highlighting, edit-state management. Collapse is a DERIVED per-file decision (`isFileCollapsed`: userExpanded > userCollapsed > auto predicate, evaluated at render time ŌĆö there is no post-render auto-collapse effect, so big files never transiently build their bodies); offscreen bodies defer behind estimated-height placeholders mounted by an IntersectionObserver (┬▒2 viewports, one per frame, `holdViewport` re-anchoring) or by `revealFile(path)` ŌĆö the single force-mount/expand hook every programmatic jump (search, hunk/annotation nav, editor/preview open, file-tab click) must use.
