# File Guide

Per-file map of the codebase: what each file does plus the invariants, gotchas, and design notes that matter when changing it. The compact one-line-per-file index lives in [CLAUDE.md](../CLAUDE.md#project-structure); this is the detailed version. Read a file's entry before modifying the file or adding a new caller of its exports; update the entry when adding, removing, or renaming a file or when its contract changes (and keep the CLAUDE.md one-liner in sync). Test files get entries only when the test setup itself carries invariants worth noting.

See [ARCHITECTURE.md](ARCHITECTURE.md) for the system-level design these files implement.

## cmd/lightjj/

- **`main.go`** ‚Äî CLI entry point, flag parsing, embeds frontend-dist/. Startup order matters for persistence: MigrateConfigIfNeeded ‚Üí MigrateStateIfNeeded ‚Üí tab restore (ReadPersistedTabs) ‚Äî the state migration must run before tab restore or upgraded users lose their tabs on first launch.
- **`frontend_embed.go`** ‚Äî `-tags embed` build: frontendHandler serves the go:embed'd frontend-dist/.
- **`frontend_stub.go`** ‚Äî Default (no embed tag) build: frontendHandler returns a static "frontend not bundled" help page pointing at release binaries or a `-tags embed` source build, so plain `go install` binaries don't 404.
- **`session_file.go`** ‚Äî Agent port-discovery: writes {pid,addr,repo_dir,mode} to `$XDG_RUNTIME_DIR/lightjj/sessions/<pid>.json` after bind; stale-pid sweep; symlink-race-hardened TempDir fallback (verifyOwnedDir double-check). No-op on Windows. Agents match by repo_dir instead of needing the random port.
- **`session_file_unix.go`** ‚Äî verifyOwner: checks the session dir is owned by the current uid (unix builds).
- **`session_file_other.go`** ‚Äî verifyOwner no-op stub for non-unix builds (Windows short-circuits earlier in writeSessionFile).
- **`api_cmd.go`** ‚Äî `lightjj api METHOD PATH [BODY]` + `lightjj sessions` subcommands: loopback-only HTTP client (validateAddr rejects non-localhost) with session-file discovery, for agent harnesses whose sandboxes denylist curl/wget. Security model + discovery algorithm in [design-notes/api-cli.md](design-notes/api-cli.md).
- **`skill_cmd.go`** ‚Äî `lightjj skill [install]`: prints or installs the go:embed'd SKILL.md (the agent-facing guide to `lightjj api`); install refuses to overwrite a file that doesn't carry the lightjj marker unless `--force`.
- **`apply_hunks.go`** ‚Äî `--apply-hunks=<spec.json>` re-entry path for `jj split --tool lightjj-hunks`: dumb writer applying the frontend-synthesized write/revert/delete spec into jj's $right dir, with path-escape checks. All hunk-patching logic lives in frontend hunk-apply.ts.

## internal/runner/

CommandRunner interface + implementations

- **`runner.go`** ‚Äî Interface definition (Run, RunWithInput, RunForMutation, StreamCombined, RunRaw, WriteFile)
- **`local.go`** ‚Äî LocalRunner: exec("jj", args) with configurable Binary. WriteFile does symlink-escape hardening (EvalSymlinks on parent + Lstat leaf) before os.WriteFile. ResolveWorkspaceRoot rejects forgotten workspaces: `jj workspace root` exits 0 inside a forgotten workspace's dir, so a `jj log -r @` probe (probeWorkspace) follows it ‚Äî tab create 400s and the startup restore loop log-and-skips instead of mounting a permanently broken tab.

## internal/api/

HTTP handlers

- **`server.go`** ‚Äî Route registration, runMutation, op-id caching, workspace store reader, helpers. For pure decode‚Üívalidate‚Üíbuild mutations the route line IS the handler (`reg("POST /api/new", mutation(s, newRequest.validate, newRequest.build))`); the endpoint table in docs/ARCHITECTURE.md is drift-checked by architecture_doc_test.go. `Server.OpenTabRoots` (func field, injected by TabManager.addLocked; nil in tests/standalone) reports all open tab roots for cross-tab guards. The op-id cache is encapsulated behind `getOpId()`/`setOpId()`/`casOpId()` ‚Äî the `cachedOp`/`cachedMu` fields are an implementation detail nothing else (besides `refreshOpId`, which composes them with the actual jj read) should touch; the SSH poll loop's compare-and-swap goes through `casOpId`, never the mutex directly. `trySnapshot()` is the pause-respecting snapshot path used by snapshotLoop and handleFileWrite; handleSnapshot checks the pause inline instead because it needs runMutation's response envelope and runMutation itself bumps the pause counter.
- **`handlers.go`** ‚Äî Endpoint implementations, flag validation, and `mutation[Req](s, validate, build)` ‚Äî the generic decode‚Üívalidate‚Üíbuild‚ÜírunMutation factory. Pure mutations exist as request structs with `validate()`/`build()` method pairs (no named handler func); `opMutation`/`bookmarkMutation`/`bookmarkRevMutation`/`bookmarkRemoteMutation` are thin adapters over `mutation()` for families sharing one request shape. Hand-rolled handlers are the ones with extra behavior: stdin (describe), streaming (git push/fetch/alias/index-paths), runner-backed validation (alias), tempfiles (split-hunks, merge-resolve), non-400 guards (workspace forget), server-state builds (workspace add), post-mutation side effects (snapshot, file-write, unlock-repo). Every success response uses the MutationResult envelope `{output, warnings?}` (apiVersion 2 unified the legacy `{ok}`/`{status}` shapes). Workspace name validation is two-tier: `workspaceNameRe` (strict, ADD only ‚Äî name becomes a dir suffix) vs `validateWorkspaceRefName` (loose, forget/rename ‚Äî jj allows dots/slashes in existing names; flag injection blocked by the `--` in the command builders). `workspaceNameMutation(build, guard)` is the forget/rename factory (decode‚Üívalidate‚Üíguard‚ÜírunMutation; stays hand-rolled because guard returns non-400 statuses); `forgetOpenTabGuard` 409s when the target workspace's path (via `workspacePathMap`, shared with handleWorkspaces) matches any open tab root, and allows when the path can't be resolved.
- **`handlers_test.go`** ‚Äî Handler tests with MockRunner
- **`architecture_doc_test.go`** ‚Äî Drift guard for the docs/ARCHITECTURE.md endpoint table: parses `| METHOD | \`/api/...\` |` rows and asserts two-way sync with `routes()` (every registered route documented, every documented `/api` row registered). Tab routes (`/tabs`, `/tab/{id}/*`) are exempt ‚Äî they're TabManager's, not `routes()`'. Sibling of agent_docs_test.go (the agent_api.md guard).
- **`integration_test.go`** ‚Äî Integration tests (build-tagged)
- **`watcher.go`** ‚Äî fsnotify on .jj/repo/op_heads/heads/ + SSE push, periodic util snapshot. Internals are typed: broadcasts are `sseEvent{name, data, droppable}` values (constructors `opEvent`/`navEvent`/`pollFailEvent`, sentinel vars `evStaleWC`/`evFreshWC`/`evPollOk`) that map 1:1 onto the `event:`/`data:` wire frames ‚Äî the names/payload shapes are the api.ts contract and must not change; `handleEvents` is a single formatted write with zero event-type dispatch. `broadcast()` drops `droppable` events (op-ids, nav hints) when a subscriber's buffer is full, but EVICTS the oldest queued event to deliver non-droppable sentinels (stale-wc/fresh-wc, pollfail/pollok) ‚Äî they're edge-triggered and never re-sent, so a dropped one would stick the client until reconnect. Both background loops classify probe results through ONE shared `probeTracker` (`ok`/`staleWC`/`fail`): it owns the consecutive-error counter, the 3rd-then-every-12th log throttle, the `pollFailThreshold` UI gate, and the unconditional poll-fail-clear discipline (the stale-WC-interlude-leaves-pollFailed-stuck bug previously had to be fixed once per loop ‚Äî the invariant comments now live once, on the struct). SSH mode uses `sshPollLoop` (`PollOpId` ‚Äî implicit snapshot + op-id in one round trip; no remote deps; `Server.casOpId` CAS against the pre-poll cache value so a concurrent runMutation advance isn't regressed). `interval <= 0` gates both loops (parity). Stale-WC detection: `isStaleWCError()` ‚Üí `setStale(bool)` (staleMu-serialized Swap+broadcast ‚Äî handler clear can't race a loop set and emit sentinels out-of-order; Swap edges don't re-fire so the race has no self-heal) ‚Üí `evStaleWC`/`evFreshWC` ‚Üí frontend non-dismissable warning with "Update stale" action. Both handleSnapshot + handleWorkspaceUpdateStale call `clearStale()`. Snapshot-pause counters are shared per canonical repo path via the package-level `pauseCounterFor` registry, resolved at watcher CONSTRUCTION (`pauseKey`: RepoDir locally, RepoPath over SSH) ‚Äî same-path tabs share one counter with no post-construction field swapping. `handleEvents` refreshes the op-id if the cache is empty (read-only session path), emits current stale + poll-fail state unconditionally on connect (both branches ‚Äî false branch covers SSE-dropped-during-staleness ‚Üí CLI-recovered ‚Üí reconnect), per-write WriteDeadline extension (not blanket disable ‚Äî dead TCP surfaces in ‚â§85s).
- **`watcher_test.go`** ‚Äî Broadcaster tests (subscribe/broadcast, droppable-drop on full buffer, sentinel eviction on full buffer), sshPollLoop tests via seqRunner (hook field for mid-call injection), setStale edge-only invariant, handleEvents refresh-if-empty table, CAS-guards-concurrent-advance, stale-interlude-clears-pollfail on BOTH loops (the shared probeTracker discipline), shared-pause-counter-by-path
- **`tabs.go`** ‚Äî TabManager: per-tab Server + Watcher mounted at /tab/{id}/; TabResolve + TabFactory injection (mode-agnostic validation via closure); openTabs persisted to **state.json** via `SetOpenTabs` (state.go ‚Äî NOT config.json, so tab churn never runs the JSONC patch machinery; tab 0 excluded ‚Äî fights with -R on next launch); startup restore re-runs resolve() so moved repos log-and-skip. `OpenTabRoots()` (RLock) returns every open tab's root; addLocked injects it into each mounted Server (feeds the workspace-forget cross-tab guard). NewTabManager also registers the host-scoped config + state routes on its Mux. addLocked wires only the SSE subscriber hooks (idle shutdown); snapshot-pause sharing across same-path tabs is NOT wired here ‚Äî watchers resolve their shared counter at construction (`pauseCounterFor`, watcher.go).
- **`config.go`** ‚Äî Server-side JSONC config (tailscale/hujson) for HUMAN-edited keys only; mergeAndWriteConfig applies per-key RFC-6902 add patches so comments on existing keys survive; fresh installs (ENOENT) seed from configTemplate, corrupt files NEVER reseed ‚Äî 422 + frontend warning keeps user's bad file untouched (ErrConfigUnparseable sentinel). Machine-written keys (openTabs, recentActions) live in state.json (state.go), never here. GET /api/config returns Standardized JSON (422 on corrupt); GET/POST /api/config/raw round-trips raw JSONC for ConfigModal (text/plain only, 1MB cap, isLocalOrigin guard). Atomic writes go through the shared `atomicWriteFile` (jsonstore.go).
- **`config_jsonc.go`** ‚Äî hujson helpers: standardizeJSONC (bytes.Clone first ‚Äî hujson.Standardize mutates input via aliased Extra slices), unmarshalJSONC, patchConfigKeys, removeConfigKeys (RFC-6902 "remove"; only pass keys that exist ‚Äî removing an absent path errors). The centralised "Standardize before Unmarshal" idiom ‚Äî any new json.Unmarshal on config bytes must go through unmarshalJSONC or break the first time the user adds a comment.
- **`config_template.go`** ‚Äî First-run JSONC template string (in-tree constant, not a file). Teaching comments over theme, fontSize, editorArgs, font families. Kept in sync with frontend defaults and [CONFIG.md](CONFIG.md) by convention ‚Äî see the top-of-file comment.
- **`state.go`** ‚Äî Machine-state store: state.json (plain JSON, NOT JSONC) next to config.json, holding values lightjj writes itself ‚Äî `openTabs` (PersistedTab; SetOpenTabs filter-merges by (mode,host) so concurrent sessions don't stomp each other; ReadPersistedTabs for startup restore) and `recentActions` (namespace ‚Üí key ‚Üí ms timestamp; SetRecentActions whole-section replace). One `stateMu` over every read-modify-write; atomic writes via `atomicWriteFile`. Missing/zero-byte/corrupt files all read as fresh (machine state is recoverable; nothing to protect with a 422). `MigrateStateIfNeeded` (main.go startup, after MigrateConfigIfNeeded, before tab restore) imports legacy openTabs/recentActions from config.json then removes them via removeConfigKeys ‚Äî import is written BEFORE removal so failure can't lose data. HTTP: GET/POST /api/state/recent-actions (package-level handlers, host-scoped like config; POST has the isLocalOrigin guard).
- **`jsonstore.go`** ‚Äî Generic `jsonCollection[T hasID]` keyed flat-file JSON-array store (key ‚Üí one file via `pathFor`, items addressed by ID). Owns the machinery annotations + doc-comments previously hand-mirrored: per-collection read-modify-write mutex, merge-on-upsert (preserve-on-empty, runs BEFORE stamp so merge detects "client omitted X" by zero value), server-side `stamp` defaults, configurable `deleteMatch` (cascade), delete-file-when-last-item-removed, and `UpsertBatch` (whole batch under one lock). Collections MUST be package-level vars, not Server fields ‚Äî store files are shared across tabs, so the mutex must be too. Errors are `storeError{status,msg}`; handlers map via `storeErrStatus` (bad key ‚Üí 400, I/O ‚Üí 500). Low-level readJSONStore/atomicWriteJSON live here too, plus `atomicWriteFile` ‚Äî the SINGLE temp-file+rename primitive shared by jsonstore/config/state (it prevents torn writes only; lost-update prevention is the caller's mutex).
- **`annotations.go`** ‚Äî Per-changeId review comments (annotations/{changeId}.json): `Annotation` type + `annotationStore` collection config (pathFor=annotationsPath with changeId regex validation; merge preserves resolution/resolvedAtCommitId AND legacy Status under the same condition, plus createdAt; stamp fills createdAt) + thin GET/POST/DELETE handlers. DELETE without id = clear-all (DeleteAll).
- **`doc_comments.go`** ‚Äî Doc-mode range-anchored comments/suggestions: `DocComment`/`DocAnchor` types + `docCommentStore` collection config (key = RepoPath+"|"+cleanDocPath(filePath) via `docCommentKey`, filename = sha256(key)[:16]; merge preserves resolution/resolvedAt/createdAt; stamp fills randID + createdAt; deleteMatch cascades to replies via ParentId) + GET/POST/DELETE/batch handlers. cleanDocPath: ToSlash+Clean+strip ./, reject ../ and / (400). POST validates anchor.selection. DELETE requires id (no clear-all). Batch validates ALL comments before writing ANY (all-or-nothing via UpsertBatch).
- **`agent_docs.go`** ‚Äî `GET /api/agent` serves go:embed agent_api.md (the agent-facing contract). agent_docs_test.go fetches it and asserts every /api/* path mentioned is a registered route ‚Äî the structural guard against doc/route drift.
- **`symbol.go`** ‚Äî rg-backed go-to-definition for `GET /api/symbol` (Tier 1 ‚Äî heuristic, no language server): per-language definition regex run against the working copy via `rg --json` through RunRaw (so it works in SSH mode), identifier-gated `name` param, capped hits with leading doc-comment context.
- **`focus.go`** ‚Äî `GET/POST /api/focus`: frontend-reported "what is the user looking at" (change_id/commit_id/active view/doc path) for the agent steering loop. Server-stamped UpdatedAt, per-tab singleton (each tab is its own Server), field-length caps.
- **`open.go`** ‚Äî Open-in-$EDITOR: buildEditorArgv ({file}/{line} substitution, argv[0] validation), handleOpenFile. Setsid via build-tagged detachProcess (open_unix.go / open_windows.go)
- **`open_unix.go`** ‚Äî detachProcess via Setsid (!windows) so Ctrl+C on lightjj doesn't kill the spawned editor.
- **`open_windows.go`** ‚Äî detachProcess no-op (windows builds).
- **`gzip.go`** ‚Äî Gzip response middleware (lazy-init writer, Flush passthrough for SSE)

## frontend/src/lib/

- **`api.ts`** ‚Äî Typed API client, op-id tracking, commit_id-keyed LRU cache. `wireAutoRefresh` SSE reconnects on backend restart via `everSawEvent` heuristic (mirrors Go's `everSawLine`): handleEvents pushes op-id immediately on connect, so CLOSED-before-event = watcher absent ‚Üí give up; CLOSED-after-event = transient ‚Üí backoff retry. `resolvedInfo()` exposes the memoized /api/info value synchronously (set when the `_info` promise resolves, cleared with the session memos) ‚Äî jj-features.svelte.ts reads the backend `features` map from it in the same tick App calls setDetectedJJVersion. All mutation methods return `MutationResult` (`{output, warnings?}`) ‚Äî including fileWrite/openFile/unlockRepo since the apiVersion-2 envelope unification. Machine-state methods `recentActions`/`saveRecentActions` (+ `RecentActionsState` type) hit /api/state/recent-actions ‚Äî host-scoped data, but routed through the normal tab-scoped request() since every Server registers the state routes.
- **`RevisionGraph.svelte`** ‚Äî Revision list with graph gutter rendering; JS-tracked hoveredIndex (mousemove-driven, not :hover). Virtualized via `createWindower` (virtual.svelte.ts) above VIRTUALIZE_THRESHOLD=150 flatLines (~50 commits). Fixed 18px rows ‚Üí pure-arithmetic windowing (no per-row measurement). Row template is a {#snippet} shared by virtual/eager paths. Pure display ‚Äî no revset filter input (that lives in App.svelte above RevisionGraph inside `revision-panel-wrapper`). All context-menu emitters (row AND workspace badges) share the `anyModeActive || isRefreshing` suppression gate ‚Äî new right-click surfaces must add it too.
- **`virtual.svelte.ts`** ‚Äî `createWindower()` fixed-row-height virtualization. count/scrollEl passed as getters ‚Üí reactive without $effect glue. `items` is $derived reading count directly ‚Üí no lag-frame between count shrink and item re-render (the tanstack setOptions-via-$effect problem). `scrollToIndex` implements align:'auto'. `holdViewport(scrollEl, anchorSel, fn, gen?)` runs a layout-shifting mutation without the line under the user's eye moving: capture first `anchorSel` match with top ‚â• container-top ‚Üí fn() ‚Üí `await tick()` ‚Üí correct scrollTop by delta (tick = post-DOM-flush pre-paint; rAF would paint shifted first). `anchorSel` MUST be per-row (e.g. `.diff-file`) ‚Äî a wrapper div's top doesn't move when its contents reflow. gen() lets a concurrent scrollTo win.
- **`GraphSvg.svelte`** ‚Äî SVG renderer for graph gutter characters (pipes, curves, node dots)
- **`DiffPanel.svelte`** ‚Äî Diff viewer: unified/split toggle, syntax highlighting, edit-state management. Collapse is a DERIVED per-file decision (`isFileCollapsed`: userExpanded > userCollapsed > auto predicate, evaluated at render time ‚Äî there is no post-render auto-collapse effect, so big files never transiently build their bodies); offscreen bodies defer behind estimated-height placeholders mounted by an IntersectionObserver (¬±2 viewports, one per frame, `holdViewport` re-anchoring) or by `revealFile(path)` ‚Äî the single force-mount/expand hook every programmatic jump (search, hunk/annotation nav, editor/preview open, file-tab click) must use. Compute caps (HIGHLIGHT_SKIP_LINE_LIMIT/WORD_DIFF_LINE_LIMIT, 5k lines) are independent of the collapse thresholds. Reset effect (activeRevisionId change) clears edit/search/expanded state + saves/restores collapseStateCache (change_id-keyed). startEdit/discardFile guard against navigation-during-await by comparing captured changeId vs LIVE diffTarget prop. **Conflict resolution never moves @ in local mode**: `quickResolve(path, 'ours'|'theirs')` (one-click whole-file resolve) and `saveMerge` both write through `writeResolution` ‚Üí `resolveConflictFile` (conflict-resolve.ts: @ ‚Üí fileWrite; non-@ local ‚Üí mergeResolve; non-@ SSH ‚Üí explicit jj-edit fallback surfaced in the edit-error banner as "working copy moved") ‚Äî NOT planTake's incremental surgery, so quick-resolve sidesteps the planTake blank-line gap. `fetchFileForEdit(path, prefix, moveWorkingCopy)` is the shared fetch prologue: `moveWorkingCopy=true` (auto-`jj edit` non-`@`) ONLY on editor-opening paths ‚Äî `startEdit` and `openEditorFallback` (the N-way/git-style/auto-resolved-race fallback, which opens the raw FileEditor whose Save writes to @); resolution paths pass `false` (pure read). An EMPTY chosen side in quickResolve (modify/delete ‚Äî jj materializes the deleted side as empty) is refused with an editError instead of written, since the resolution write has no delete path and a zero-byte write would mis-resolve the tree. `toggleSplitView()` is EXPORTED and is the only sanctioned way to flip split/unified from outside (App's Cmd+K palette routes through it) ‚Äî it confirms before discarding an open FileEditor buffer; writing `config.splitView` directly bypasses that confirm. `[`/`]` ‚Üí `stepHunk(dir)` over `flatHunks` (cross-file); `{`/`}` ‚Üí `stepAnnotation(dir)` over `navAnnotations` (file-order sorted). Both via `scrollToHunk(path, hunkIdx)` (revealFile ‚Üí rAF ‚Üí data-attr query). `diffContentKey` is a REQUIRED prop: `contentMatchesTarget` is the strict equality `diffContentKey === activeRevisionId` (no `!diffPending` fallback) ‚Äî every mount must pass it (App passes `nav.diffContentKey`; test fixtures default it to `diffTargetKey(diffTarget)`).
- **`diff-cache.ts`** ‚Äî App-lifetime caches: derivedCache (highlight+word-diff, commit_id-keyed), parsedDiffCache (diff-text-keyed for ref-identity on revisit), collapseStateCache (change_id-keyed `CollapseMemo` = the user's explicit `{collapsed, expanded}` intent sets ‚Äî the auto-collapse predicate is re-evaluated live, never persisted). Cursor/hover/j/k via createListCursor (hoverMovesCursor; **no Enter/Escape hooks** ‚Äî those must bubble to App's global handler which executes/cancels the inline mode); Space/a/n layered in handleKeydown. - **`markdown-render.ts`** ‚Äî marked (GFM) + DOMPurify. `renderMarkdown(src, ctx?, stampSrc?)` is sync; per-render mutable state (pendingDiagrams/imgCtx/footnotes/fnLabelToIdx/headingSlugs/stamp) bundled into ONE `rs: RenderState` slot ‚Äî hooks read `rs!.x` (non-null inside the try/finally). `fnScope` stays module-level (cross-render counter). `stampedBlocks(container, totalLines)` ‚Üí `{el, start, end}[]` for MarkdownPreview's gutter measurement.
- **`mermaid.ts`** ‚Äî `ensureMermaidLoaded`/`tryRenderDiagram`/`wirePanzoom` ‚Äî beautiful-mermaid lazy-load + render. Extracted from markdown-render so pm-mermaid.ts (doc-mode) imports the renderer without pulling marked + DOMPurify into its chunk.

- **`excalidraw-render.ts`** ‚Äî PURE ‚Äî `renderExcalidrawSVG(jsonText)` ‚Üí SVG string | null. Zero-dep clean-geometry renderer (no rough.js). rectangle/ellipse/diamond/line/arrow/text/freedraw; placeholder boxes for image/frame/embeddable. `escapeAttr` (NOT escapeHtml ‚Äî `"` breaks out of attribute context) on color/viewBackgroundColor.

- **`time-format.ts`** ‚Äî relativeTime(): compact relative ages from jj timestamps or epoch-ms (parse memoized ‚Äî same string always maps to the same ms), plus firstLine(). Shared by RevisionGraph, FileHistory*, CommentCard.
- **`scroll-into-view.ts`** ‚Äî scrollIdxIntoView(): scrolls a list row located by `data-idx` attribute, not by selected-class (a synchronous querySelector after a $state write would find the OLD row). createListCursor wraps it; direct callers are the lists that don't use the factory (SearchResults, FileHistoryRail).
- **`list-cursor.svelte.ts`** ‚Äî `createListCursor()` keyboard-list cursor factory: cursor + hovered $state, j/k/Arrow routing with the filter-input guard (j/k type while the filter is focused; arrows still navigate), Enter/Escape/'/' hooks (absent hook = key falls through unprevented), onNav (pre-move, fires even when clamped ‚Äî confirm disarm) vs onMove (actual change only), bounds-clamp $effect (must be created during component init; tests wrap in `$effect.root`), delegated `[data-idx]` mousemove hover with two modes (hoverMovesCursor for modals: one highlight serves both input methods; separate `hovered` for panels: distinct .hovered class), and data-idx scroll-into-view. `handleKey(e): boolean` supports both self-focused lists (element onkeydown) and delegated-key lists (exported handleKeydown App calls); domain keys layer behind `if (cursor.handleKey(e)) return`. Direct `cursor.index =` writes never fire onMove/scroll ‚Äî restore/reset effects use them deliberately.
- **`loader.svelte.ts`** ‚Äî createLoader() async factory with generation counter
- **`revision-navigator.svelte.ts`** ‚Äî createRevisionNavigator(): diff/files/description loaders + revGen batch-fetch orchestration + navigateCached (double-rAF paint-first deferral) / navigateDeferred (50ms debounce coalesce). cancel() clears both timers + bumps revGen ‚Äî switchToLogView/onDestroy/handleRevsetSubmit all call it. **Sole owner of `loadedTarget`/`diffContentKey`/`diff.value` ‚Äî every diff load goes through `loadDiffAndFiles`/`applyCacheHit`/`loadMulti` so the triple advances atomically.** App calling `diff.load()` directly desyncs DiffPanel's `contentMatchesTarget` (the v1.19‚Üí1.20 memo-poisoning class). Scheduling was previously inline in App.svelte's selectRevision; moved here so the "rapid j/k only renders destination" contract is testable with fake timers.

- **`keyboard-gate.ts`** ‚Äî PURE ‚Äî routeKeydown(ctx, handlers) gate-priority router extracted from App.svelte's handleKeydown. Tests lock the ORDER invariants (globalOverrides before inInput, inlineNav swallows all, etc).
- **`modes.svelte.ts`** ‚Äî Rebase/squash/split mode state factories. `ModeBase.diffFollows` is the per-mode nav semantic (rebase=true destination-preview, squash/split=false frozen-on-source) ‚Äî App derives `diffFrozen` from this so mouse onselect + the multi-check $effect read one source instead of each spelling out `squash.active || split.active`. `inlineMode` stays for binary gates; `diffFrozen` is the answer `inlineMode` can't give. SplitMode.review distinguishes 'v' (review: accepted/rejected labels) from 's' (split: stays/moves labels) ‚Äî same jj split underneath.
- **`slide.ts`** ‚Äî computeSlide(): Shift+J/K single-step commit reorder along a linear graph segment. Targets the topological parent/child edge (not the visually adjacent row); merge/fork points and immutable neighbors return `{ok: false, reason}` instead of a rebase target.
- **`config.svelte.ts`** ‚Äî Reactive config singleton ‚Äî primary storage os.UserConfigDir()/lightjj/config.json, localStorage as write-through cache. **Dirty-key saves**: public setters go through `setKey` which marks the key dirty; the debounced flush POSTs ONLY dirty keys (backend merges per-key) ‚Äî never the full snapshot, so two lightjj instances sharing one config.json can't revert each other's changes (lost-update fix). `applyPartial()` (hydration, ConfigModal live-apply, storage-event sync) writes state WITHOUT marking dirty ‚Üí no echo POST. `recentActions` getter/setter is a back-compat delegation to recent-actions.svelte.ts (machine state, not config ‚Äî never POSTed to /api/config). `fontSize` getter clamps to `FONT_SIZE_MIN..MAX` with `Number()`+`isFinite` so all read sites (CSS var, palette label, ¬±1 arithmetic) see a sane value regardless of load path.
- **`recent-actions.svelte.ts`** ‚Äî last-used timestamps for bookmarks (BookmarkModal/BookmarkInput recency sort), persisted in the machine-state store (state.json via api.recentActions/api.saveRecentActions ‚Äî debounced full-map POST, hydrate-once at module load with a dirty guard so late hydration can't clobber local writes). Exposes `recentActionsStore` (whole-map accessor used by config.svelte.ts's back-compat delegation) + the `recentActions(namespace)` factory. Persistence is best-effort: every api call is try/caught so partially-mocked api modules in component tests (and a down backend) degrade to in-memory-only.
- **`annotations.svelte.ts`** ‚Äî Per-line review comment store for agent workflows. createAnnotationStore() + reanchor() + exportMarkdown/JSON. Server-side storage via /api/annotations (workspace tabs share). `side?: 'old'|'new'` (default new) lets comments target deleted lines; `byLine` key includes side. Re-anchor via diffRange delta + ¬±5 content scan (new-side only ‚Äî old-side pinned to pre-image). NO negative caching of empty changeIds ‚Äî agents POST annotations out-of-band (other tabs, the agent CLI), so a "known empty" memo would hide them permanently; load() always hits the server.
